# 第 4 章练习题

## 概念题

1. 为什么公开学习资产需要主索引，而不是只放很多仓库链接？
2. 为什么“项目、源码笔记、训练输出、资料索引”适合分层组织？
3. 什么是作品的证据链？为什么它比自我描述更重要？
4. 为什么很多作品集最终失效，不是因为内容不够，而是因为结构不可持续？

## 分析题

1. 回顾你现有的 GitHub 输出，分析它们为什么还没有形成一个清晰的长期学习资产体系。
2. 一个只有项目代码、没有复盘和证据链接的仓库，在面试和协作中会缺什么？
3. 如果你准备建立一个源码阅读仓库，哪些结构设计能帮助它半年后仍然可维护？

## 代码题

1. 为你自己的作品集仓库写一个首页大纲，要求包含主索引、代表作、源码笔记、训练输出四部分。
2. 设计一个源码阅读笔记模板，方便后续持续复用。
3. 选一个项目，把 README 里的“项目简介”扩展成“项目简介 + 证据链 + 复盘入口”结构。
