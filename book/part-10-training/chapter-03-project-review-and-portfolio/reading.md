# 第 3 章延伸阅读

## 必读

- 优秀开源项目的 `README`、架构图和贡献指南
- 工程团队关于 postmortem、retrospective、design doc 的基础写作方法
- 技术求职或作品集整理中关于“如何证明能力”的高质量文章

## 进阶

- 优秀个人技术博客或源码笔记仓库，观察他们如何把学习过程沉淀成长期资产
- 高质量开源维护者的项目介绍方式，学习如何讲清结构、取舍与边界
- 面向工程师的复盘方法，例如事故复盘、项目回顾、决策记录等实践

## 实战

- 选 3 个你欣赏的 GitHub 项目，拆解它们的 `README` 结构和证据呈现方式
- 为自己建立一个统一的作品集仓库索引页，把项目、源码笔记、面试题、学习计划连起来
- 把一个旧项目重新整理后，尝试用 3 分钟口头讲述它，检验你是否真的完成了资产化
