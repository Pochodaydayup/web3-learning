# 第 5 章项目实践

## 实践目标

这次实践不是写更多代码，而是搭建你自己的长期学习操作系统。目标是让前面整本书里的内容可以持续进入复利循环。

## 操作步骤

1. 写下你当前的学习入口：
   - 在读什么
   - 在做什么项目
   - 在输出什么
   - 在复盘什么
2. 找出最容易断掉的一环。
3. 重设计一个未来四周的最小系统，要求包含：
   - 固定输入节奏
   - 固定项目推进节奏
   - 固定输出动作
   - 固定周复盘
4. 最后写一页调整说明：
   - 为什么你原来的系统不稳
   - 这次重构改了什么
   - 如何判断新系统正在复利

## 交付物

你应该交付：

1. 一份学习系统现状诊断
2. 一份四周节奏表
3. 一份长期复利判断标准
