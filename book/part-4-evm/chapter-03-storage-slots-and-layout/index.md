# 第 3 章：存储布局与 Slot

当你已经理解了调用上下文和 storage 是长期状态层之后，下一步就该真正进入 slot 布局本身。很多 Solidity 写法之所以重要，不只是语法原因，而是它们最终会映射到具体存储布局。

## 本章要解决什么

本章要解决的是“我知道 storage 很重要，但不清楚状态变量在底层如何落到 slot 上，也不知道这为什么影响升级、调试和优化”。你会建立一个足够实用的布局直觉。

## 读完后你应该会什么

- 对 slot 是怎样承载状态有更清晰的认知
- 解释简单值类型和复合结构为什么布局不同
- 理解为什么布局会影响升级安全和 gas 判断
- 带着更稳的 storage 直觉去读真实协议源码
