# 第 4 章：低级调用、返回数据与回滚

到了这一章，你会真正碰到很多“高级 Solidity 语法背后”的底层执行细节。为什么低级调用这么容易出错？为什么有些错误会冒泡回来，有些不会？为什么返回数据和 revert data 会影响上层逻辑？这些问题都在这里汇合。

## 本章要解决什么

本章要解决的是“我知道 `call`、`delegatecall`、`staticcall` 这些词，但不知道它们在工程上到底意味着什么”这个问题。你会把低级调用、返回数据、错误冒泡和回滚语义连起来理解。

## 读完后你应该会什么

- 区分常见低级调用方式在上下文与能力上的差别
- 理解返回数据与 revert data 为什么需要被认真处理
- 知道为什么低级调用常常是 bug 和安全问题高发区
- 从 EVM 视角理解“外部调用失败以后，上层代码该怎么判断”

## 本章核心概念

- **低级调用（Low-level Call）**：绕过高层接口语义，直接处理调用结果和返回数据的调用方式。
- **Return Data**：调用成功后从被调用方返回给调用方的编码结果。
- **Revert Data**：执行失败时向外层返回的错误编码信息。
- **错误冒泡（Bubbling）**：把底层调用失败信息继续向上传递给更外层调用者的行为。
