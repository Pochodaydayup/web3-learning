# 第 2 章项目实践

## 实践目标

本章实践的目标，是把一次“跨合约调用”从代码层翻译成上下文变化。你会第一次真正用“谁的代码、谁的状态、谁的 `msg.sender`”去读一段交互。

## 操作步骤

1. 写或找到一个最小 A 调 B 的合约例子。
2. 再找一个最小 proxy + implementation 的例子。
3. 对每一步调用记录：
   - 当前执行地址
   - 当前 `msg.sender`
   - 被修改的状态归属
4. 如果有条件，用 trace 把调用路径抓出来。
5. 写一页对比说明，解释 `call` 和 `delegatecall` 为什么危险面不同。

## 交付物

1. 一份调用上下文对照表。
2. 一张调用路径图。
3. 一页对比复盘。
