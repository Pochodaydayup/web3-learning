# 第 2 章：Opcode 与调用上下文

理解了 storage、memory 和 calldata 之后，下一步要补的是“谁在当前上下文里执行，执行中到底能看到什么”。这会直接影响你对 `msg.sender`、`msg.value`、外部调用、`delegatecall` 和 trace 的理解。

## 本章要解决什么

本章要解决的是“我知道 EVM 在执行，但对调用上下文和 opcode 层含义还是抽象的”。我们不会把你变成汇编工程师，但会把最值得工程师掌握的上下文概念讲清楚。

## 读完后你应该会什么

- 解释当前调用上下文包含哪些关键信息
- 理解普通 `call` 和 `delegatecall` 在上下文层面的差异
- 知道为什么 trace 和低级调用会让问题变得更透明
- 对 opcode 有一个工程上够用的认识框架
