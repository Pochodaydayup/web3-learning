# 第 6 章练习题

## 概念题

1. 为什么说部署合约本质上也是一次 EVM 执行，而不是简单存代码？
2. init code 和 runtime code 的职责分别是什么？
3. constructor 为什么只属于部署期，而不是运行时接口的一部分？
4. CREATE 和 CREATE2 在地址推导思路上有什么核心差异？

## 分析题

1. 某合约部署时 gas 消耗异常高，但运行后函数调用成本正常。请分析可能与哪段代码有关。
2. 某团队发现 constructor 参数一改，部署交易数据和最终地址预测都变了。请解释原因。
3. 一个 factory 部署出的实例地址不符合预期。请从 nonce、salt 和 init code 角度分析。

## 代码题

1. 画出一次普通合约部署的执行路径图，标出 init code、constructor、runtime code 的边界。
2. 给一个简单 factory 写结构说明，明确：
   - 谁发起创建
   - 谁接收 runtime code
   - 未来调用打到哪里
3. 审查你自己的一个部署脚本，解释其中每项部署参数影响的是创建流程的哪一段。
