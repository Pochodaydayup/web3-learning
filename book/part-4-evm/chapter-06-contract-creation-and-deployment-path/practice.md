# 第 6 章项目实践

## 实践目标

这次实践要让你把一次部署，从“会运行脚本”升级为“能解释 EVM 到底做了什么”。

## 操作步骤

1. 选择一个你熟悉的合约项目。
2. 找出它的：
   - constructor 参数
   - 部署脚本
   - 编译产物中的部署相关 bytecode
3. 画出部署路径：
   - 谁发起交易
   - init code 做什么
   - runtime code 怎样被产出
4. 如果项目里有 factory 或 proxy，再补一张扩展图。
5. 选一次历史部署失败或参数填错的例子，解释失败发生在创建流程哪里。

## 交付物

你应该交付：

1. 一张部署执行路径图
2. 一份 constructor 与 runtime code 边界说明
3. 一份部署失败定位笔记
