# 第 1 章项目实践

## 实践目标

本章实践的目标，是让你把一个真实合约重新从 EVM 资源视角读一遍。你不需要实现整个虚拟机，只需要能把 Solidity 写法重新映射回 storage、memory、calldata 和调用上下文。

## 操作步骤

1. 选一个你已经写过的小合约，例如 `Bank`、`Voting` 或 `Staking`。
2. 列出它的所有状态变量，并标记哪些属于长期主状态。
3. 检查所有外部函数参数，判断哪些天然适合 `calldata`。
4. 找一段内部处理逻辑，说明哪些数据只在执行期存在于 memory。
5. 如果项目里有外部调用，补一段调用路径说明，解释上下文什么时候切换。
6. 最后写一份“EVM 视角代码走查”，不要讲业务，只讲底层资源和执行边界。

## 交付物

1. 一份状态变量分类表。
2. 一份函数参数的数据区域说明。
3. 一份调用上下文说明。
4. 一页 EVM 视角走查笔记。
