# 第 1 章练习题

## 概念题

1. stack、memory、storage、calldata 的职责分别是什么？
2. 为什么 storage 写入比 memory 使用更值得谨慎？
3. 为什么很多外部函数参数更适合 `calldata`？
4. 调用上下文为什么会影响安全和调试？

## 分析题

1. 解释为什么“storage 贵”不应该只被背成结论，而应该反过来影响状态设计。
2. 某函数在处理输入数组时，一开始就把整个 `calldata` 数组复制进 `memory`，但后续只做遍历读取。请分析这种设计。
3. 为什么理解 EVM 的调用上下文，会帮助你更好地理解重入问题？

## 代码题

1. 找一个你自己的 Solidity 合约，标出其中哪些变量最终对应 storage，哪些参数是 calldata，哪些中间结构在 memory 中存在。
2. 写一个接受数组输入的函数，分别用 `memory` 和 `calldata` 两种方式实现，并比较你的理解差异。
3. 结合一次失败交易的 trace，尝试写一段说明，解释执行是在哪个阶段触发 revert 的。
