# 第 3 章：阅读 Uniswap V2

到了这一卷，真正重要的已经不是“再学一个新概念”，而是把前面学过的状态、事件、价格、调用流程和安全判断，第一次放到一个真实经典协议里。Uniswap V2 非常适合作为源码阅读训练的第一个中型协议。

## 本章要解决什么

本章要解决的是“面对一个真正有影响力的协议，我该从哪里开始读，才能既不迷路，也不只停留在表面函数名”。你会围绕 Pair 的状态、流动性添加移除、swap 路径和价格约束，建立一条核心阅读主线。

## 读完后你应该会什么

- 知道 Uniswap V2 应该先读哪些文件，为什么
- 理解 Pair 合约里最关键的状态是什么
- 能讲清一次 swap 背后的状态变化和约束关系
- 把源码阅读从“逐行翻译”升级为“抓结构、抓不变量、抓资金流”

## 本章核心概念

- **储备记账（Reserve Accounting）**：协议如何记录池子里资产数量及其变化的会计逻辑。
- **k 不变量（k Invariant）**：Uniswap V2 用乘积关系约束池子状态的核心不变量。
- **手续费路径（Fee Path）**：交易手续费在协议内部如何计入、累积和分配的流程。
- **LP Mint / Burn**：流动性份额如何被铸造和销毁的逻辑。
