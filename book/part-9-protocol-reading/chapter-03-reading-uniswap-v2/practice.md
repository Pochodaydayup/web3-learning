# 第 3 章项目实践

## 实践目标

这次实践要把“读源码”变成真正的输出。你需要围绕 Uniswap V2 Pair 产出一份属于你自己的结构化阅读笔记，而不是停留在浏览器书签和零散高亮上。

## 操作步骤

1. 找到 Uniswap V2 Core 仓库，定位 `Factory` 与 `Pair`。
2. 先只做结构笔记：
   - 主要文件有哪些
   - 每个文件扮演什么角色
   - 核心状态集中在哪个合约
3. 第二轮只读 Pair：
   - 状态变量
   - `mint`
   - `burn`
   - `swap`
4. 画出一张图，说明：
   - 资产如何进入池子
   - 资产如何离开池子
   - reserves 何时更新
   - 事件何时发出
5. 最后写一页“如果我要从零实现一个最小 AMM，我会优先保留什么”的总结。

## 交付物

你应该交付：

1. 一份结构图
2. 一份 Pair 核心状态笔记
3. 一份 `swap` 解释文档，要求能讲给别人听
