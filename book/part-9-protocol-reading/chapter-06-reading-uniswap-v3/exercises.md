# 第 6 章练习题

## 概念题

1. 为什么说 Uniswap V3 的复杂度不只是“V2 多了一些细节”，而是流动性表达模型改变了？
2. 为什么阅读 V3 时，第一张图更适合画价格区间与头寸关系，而不是直接画函数调用图？
3. 为什么价格移动与 tick 穿越是理解 V3 状态变化的关键？
4. 读复杂协议时，为什么应先抓结构再逐步压数学细节？

## 分析题

1. 某同学一开始就去读 V3 的数学库，结果越读越乱。请分析他可能跳过了哪些必要阅读层。
2. 如果你把 LP 头寸还当作 V2 那种“共享统一池子份额”来理解，会在哪些地方误判 V3 设计？
3. 读 V3 时，为什么只顺着文件树线性浏览往往效率不高？

## 代码题

1. 画一张 V3 阅读路线图，至少包含概念层、状态层、关键路径层和数学层。
2. 为 V3 的 `mint / burn / swap` 三条主线写一页“先看什么、后看什么”的阅读说明。
3. 选一个你还没读透的协议，套用本章方法写一版阅读骨架。
