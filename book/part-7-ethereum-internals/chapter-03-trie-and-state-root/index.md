# 第 3 章：Trie 与 State Root

很多开发者学到底层时，一看到 `stateRoot`、`storage proof`、`Merkle Patricia Trie` 就会下意识后退，觉得这些概念离日常开发太远。其实不必把它们想得过于神秘。你只要先明白：链为什么需要一种可以高效承诺全局状态、又能局部验证状态的结构，这一章就有了入口。

## 本章要解决什么

本章要解决的是“区块头里那个 `stateRoot` 到底代表什么，以及它为什么能成为全局状态的摘要”。你会用工程视角理解 trie，不追求数学证明，但追求能把账户状态、存储槽和状态根串成一个可解释闭环。

## 读完后你应该会什么

- 说清 `stateRoot` 为什么不是某个简单哈希，而是整棵状态结构的承诺
- 理解账户状态和合约存储为什么都能被组织进 trie 体系
- 知道 proof 的存在意义，以及为什么节点不必把全部数据都发给你
- 把区块头、账户查询、storage slot 和状态证明连起来理解
