# 第 3 章项目实践

## 实践目标

这次实践要把抽象的 `stateRoot` 变成你能解释给别人的东西。你需要围绕一个真实地址或合约，完成一份“状态证明入门说明书”。

## 操作步骤

1. 选一个目标：
   - 一个普通账户余额
   - 一个 ERC20 合约的某个 `balanceOf`
   - 一个简单合约的单个状态变量
2. 查出对应区块头中的 `stateRoot`。
3. 如果节点支持，调用 `eth_getProof` 获取账户或 slot 证明。
4. 用自然语言写清楚：
   - 这个状态值属于哪一层
   - 为什么它能被纳入某个 root
   - 证明路径在逻辑上起什么作用
5. 最后补一张图，把账户状态、storage root 和 slot value 的关系串起来。

## 交付物

你应该产出：

1. 一份带图的笔记
2. 一组真实 RPC 返回样例
3. 一段你自己的解释：面向“已经会写 dApp，但不懂 state root 的开发者”说明这套结构
