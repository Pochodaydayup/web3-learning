# 第 1 章练习题

## 概念题

1. nonce 对账户交易有什么作用？
2. 为什么 mempool 不是结果区？
3. receipt 和主状态分别解决什么问题？
4. state root 为什么重要？

## 分析题

1. 某用户连续发送两笔交易，第二笔迟迟不上链。请从 nonce 和排序角度解释可能原因。
2. 为什么拿到交易哈希不能代表业务已经成功？
3. 如果前端只依赖 event，不再确认主状态，会在哪些场景下踩坑？

## 代码题

1. 选一笔真实交易，记录它的 nonce、gas 参数、receipt 状态和 logs。
2. 写一页说明，解释这笔交易从广播到状态确认经历了哪些阶段。
3. 用你熟悉的 RPC 工具读取某个区块的基础字段，并写出它们在执行流程里的位置。
