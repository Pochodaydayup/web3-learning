# 第 1 章：交易、区块与状态

如果你能把一笔交易从用户点击开始，一直讲到 state root 和 receipt 结束，你对 Ethereum 的理解已经明显超过很多只会“调接口”的开发者。本章就是帮你建立这条闭环。

## 本章要解决什么

本章要解决的是“这些底层名词听过很多，但我讲不出它们是怎样串起来的”。我们会把交易对象、账户 nonce、mempool、区块打包、EVM 执行、receipt 和状态根连成一条连贯链路。

## 读完后你应该会什么

- 解释一笔交易从广播到打包的流程
- 说明 nonce 在排序和替换里的作用
- 知道 receipt 和 state root 分别站在哪个结果层次
- 把区块浏览器里常见字段重新映射回真实底层含义
