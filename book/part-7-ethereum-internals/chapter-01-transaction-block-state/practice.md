# 第 1 章项目实践

## 实践目标

本章实践的目标，是让你真正读懂一笔真实链上交易的“全身报告”，而不是只看一个 success 标记。你会把交易对象、区块信息、receipt、logs 和状态变化串起来。

## 操作步骤

1. 选择一笔你自己发出的测试网或主网交易。
2. 记录它的基础信息：
   - 发送账户
   - nonce
   - to
   - value
   - input data
3. 在浏览器或脚本中查看它最终进入的区块。
4. 读取 receipt，确认执行状态和 gas 使用。
5. 如果有事件，记录 logs 的主题和主要参数。
6. 再次读取合约主状态，验证这笔交易到底改变了什么。
7. 输出一份完整复盘，按时间顺序描述整条链路。

## 交付物

1. 一份交易字段清单。
2. 一份区块与 receipt 说明。
3. 一页“交易、区块与状态”复盘文档。
