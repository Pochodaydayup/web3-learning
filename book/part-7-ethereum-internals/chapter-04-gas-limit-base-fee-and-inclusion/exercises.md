# 第 4 章练习题

## 概念题

1. gas limit 和实际 gas used 的关系是什么？
2. base fee 与 priority fee 各自在解决什么问题？
3. 为什么同 nonce 交易可以被替换？
4. 为什么交易能否被包含，不能只看“我设置了多少总费用”？

## 分析题

1. 某笔交易长期 pending，但调用逻辑本身没有问题。请从包含经济学角度分析可能原因。
2. 为什么钱包里的“加速”操作本质上是在做 replacement？
3. 在网络拥堵时，base fee 上升会给普通用户和协议机器人分别带来什么影响？

## 代码题

1. 随机找三笔不同时间段的交易，对比它们的 fee 相关字段，写出你对包含环境的推断。
2. 解释一个 speed up 交易样本，说明原交易与替换交易的关系。
3. 为一个前端交易面板设计状态说明文案，要求能区分“等待签名”“待广播”“pending”“被替换”“已确认”。
