# 第 5 章练习题

## 概念题

1. 为什么“已进块”不等于“绝对最终不可逆”？
2. 为什么最终性既有概率属性，也有经济属性？
3. 验证者在最终性语义里扮演什么角色？
4. 为什么产品里常用确认数来表达底层最终性强弱？

## 分析题

1. 为什么大额交易和普通小额交互对确认数的要求往往不同？
2. 如果一个前端只提示“成功”而不区分确认阶段，会有什么问题？
3. 为什么说最终性不是纯技术定义，而会直接影响产品与风险决策？

## 代码题

1. 为一个钱包或 dApp 设计交易确认状态文案，区分已广播、已进块、已确认和更强最终性阶段。
2. 找一个真实交易样本，写出它从进入区块到后续确认数增长的观察记录。
3. 画一张“区块出现 -> 后续确认 -> 最终接受度增强”的流程图。
