# 第 1 章项目实践

## 实践目标

本章实践的目标，是做一次“状态成本体检”。你不需要追求极限优化，而是要学会找出一个小项目里最真实、最值得优先处理的成本问题。

## 操作步骤

1. 选择一个你已经写好的小合约。
2. 列出所有 storage 状态，并给每个状态打标签：
   - 主状态
   - 可推导状态
   - 展示辅助状态
3. 选一个高频用户函数，梳理它会读写哪些状态。
4. 判断其中是否存在：
   - 重复写入
   - 不必要状态
   - 本可使用 event 的链下展示信息
5. 尝试做一版结构性优化，然后记录你的设计变化，而不是只记录数字变化。

## 交付物

1. 一份状态成本分类表。
2. 一份高频函数的读写路径说明。
3. 一次优化前后设计对比说明。
4. 一页复盘，解释你学到的最重要 gas 判断原则。
