# 第 7 章：L1 / L2 费用模型与 Calldata 成本

前面已经从执行、存储、优化、复杂度、可维护性和测量角度讲了很多 Gas。接下来要补上一个对现代 Web3 工程越来越重要的主题：为什么你在 L2 上看到的费用结构和 L1 不一样，为什么 calldata 会成为成本核心之一，以及这些差异会怎样影响协议和前端设计。

## 本章要解决什么

本章要解决的是“为什么同样一笔交互，放在 L1 和 L2 上的成本结构差异很大，而且很多时候瓶颈不是你想象中的合约逻辑，而是数据发布成本”这个问题。你会把 L1 / L2 费用来源和 calldata 成本放进同一张性能地图。

## 读完后你应该会什么

- 理解 L1 与 L2 费用结构为什么不同
- 认识 calldata 在现代链上成本中的重要性
- 更清楚地分析 rollup 环境里的优化方向
- 在设计交互和协议时更注意数据发布成本

## 本章核心概念

- **Base Fee**：区块基础费用，是当前区块 gas 的底价要求。
- **Calldata 成本**：把输入数据带上链所需要支付的成本部分。
- **数据可用性成本（DA Cost）**：为了让交易数据可被网络验证和获取而支付的成本。
- **L2 费用模型（L2 Fee Model）**：二层网络把执行成本和数据成本组合成最终收费的方式。
