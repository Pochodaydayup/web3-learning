# 第 2 章：Storage 布局与 Packing

当你已经理解 gas 主要受状态设计影响之后，下一步就要进入一个更具体、也更容易被误学成“技巧大全”的主题：storage slot 与 packing。本章会把它讲回到资源布局和工程取舍上。

## 本章要解决什么

本章要解决的是“我知道 packing 可能省 gas，但不知道什么时候该做，什么时候不值得做”。你会从 slot、变量布局和读取/写入模式的关系出发来理解。

## 读完后你应该会什么

- 理解状态变量是怎样映射到 slot 的大致直觉
- 说明 packing 为什么能省一部分空间与成本
- 判断什么场景值得为 packing 调整布局
- 避免为了小收益把代码写得很难维护

## 本章核心概念

- **Slot**：storage 里以 32 字节为单位组织状态的逻辑单元。
- **Packing**：把多个较小变量压进同一个 storage slot 的布局方式。
- **读取放大（Read Amplification）**：为了拿到一个值而不得不附带读取更多状态的额外成本。
- **布局纪律（Layout Discipline）**：围绕状态排列、打包和扩展保持稳定规则的工程约束。
