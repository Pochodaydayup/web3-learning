# 第 1 章：安全审查基础

智能合约安全最值得优先建立的能力，不是背漏洞列表，而是建立一套稳定的审查顺序：状态会不会被重复利用，权限会不会被越过，外部调用会不会打破原有假设，外部输入是不是可信。本章会用这个顺序带你入门。

## 本章要解决什么

本章要解决的是“我知道一些漏洞名词，但不会把它们真正用到工程判断里”。我们会从最常见也最能形成直觉的几类风险开始：重入、权限控制、签名边界、`delegatecall` 和价格依赖。

## 读完后你应该会什么

- 用自己的话解释重入问题的根本原因
- 判断一个函数的权限边界是否足够明确
- 知道为什么签名不只要验地址，还要验上下文
- 对 `delegatecall`、外部价格输入和 Flash Loan 有初步风险意识

## 本章核心概念

- **Reentrancy**：外部调用尚未结束时又重新进入原逻辑路径的风险模式。
- **Access Control**：限制谁能调用哪些能力的权限控制体系。
- **CEI**：先检查、再更新状态、最后做外部交互的经典安全顺序。
- **高权限路径（Privileged Path）**：只有管理员或特殊角色才能触发的高影响力操作路径。
