# 第 1 章项目实践

## 实践目标

本章实践的目标，是给你自己的一个小合约做“第一次安全审查”。不是等代码极复杂了再想安全，而是从一开始就训练边界意识。

## 操作步骤

1. 选一个你自己写过的小合约，例如 `Bank`、`Staking`、`Voting`。
2. 列出它的核心状态和高危函数。
3. 检查每个高危函数：
   - 谁能调
   - 是否会调用外部地址
   - 状态更新在外部调用前还是后
4. 如果有签名授权逻辑，检查它是否绑定了足够上下文。
5. 如果依赖外部价格或喂价，写下你的可信假设。
6. 最后补 3 条失败路径测试，证明你的安全判断不是口头判断。

## 交付物

1. 一份安全审查清单。
2. 至少 3 条失败路径测试。
3. 一页复盘，说明你原来最容易忽略的风险面是什么。
