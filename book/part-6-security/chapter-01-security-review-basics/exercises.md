# 第 1 章练习题

## 概念题

1. 重入问题最核心的成因是什么？
2. 为什么说权限控制本质上是治理边界问题？
3. 为什么验签不能只验证签名人地址？
4. `delegatecall` 为什么天然高风险？

## 分析题

1. 某提款函数先 `call` 转账再减少用户余额。请说明攻击者可能如何利用它。
2. 某管理函数允许任意地址更新关键费率参数。请分析这一类漏洞为什么常常比复杂攻击更致命。
3. 某协议依赖单一池子即时价格做借贷清算。请分析它为什么可能被 Flash Loan 配合价格操纵利用。

## 代码题

1. 写一个最小提款函数，并给出“错误顺序版本”和“修正顺序版本”。
2. 给一个管理员函数加上 `Ownable` 或 `AccessControl` 风格的保护。
3. 设计一个签名结构体，至少包含：
   - `chainId`
   - `user`
   - `action`
   - `deadline`
   - `nonce`
