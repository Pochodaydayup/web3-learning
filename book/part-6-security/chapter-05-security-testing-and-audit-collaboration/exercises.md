# 第 5 章练习题

## 概念题

1. 为什么安全测试不能只理解成“多写几个测试用例”？
2. 安全自查清单在审计前最核心的价值是什么？
3. 为什么审计 handoff 材料不能只有源码链接？
4. 为什么说安全交付是一个闭环，而不是一个末端动作？

## 分析题

1. 一个团队只在上线前一周才第一次系统性思考安全，会面临哪些典型问题？
2. 如果测试只覆盖 happy path，却没有覆盖关键攻击假设，会带来什么风险？
3. 为什么结构化风险说明会直接影响审计意见的质量和修复效率？

## 代码题

1. 为一个简单协议写一份安全自查清单大纲。
2. 给一个项目整理“审计 handoff 资料目录”，说明每部分解决什么问题。
3. 选一个你自己的项目，写出最值得优先补的 3 个安全测试方向。
