# 第 6 章项目实践

## 实践目标

这次实践的目标，是把一个你熟悉的合约系统从“开发视角安全”扩展到“运行视角安全”。

## 操作步骤

1. 选择一个项目，可以是你自己的，也可以是你熟悉的开源协议。
2. 列出该系统上线后最重要的风险面：
   - 资产外流
   - 权限误用
   - 参数异常
   - 升级异常
3. 为每个风险面写一条最小监控规则。
4. 再写一份 first response 流程：
   - 谁接告警
   - 谁判断严重性
   - 谁能执行止损动作
5. 最后补一页事故复盘模板，明确要记录哪些事实。

## 交付物

你应该交付：

1. 一份关键风险面监控表
2. 一份 first response 流程说明
3. 一份事故复盘模板
