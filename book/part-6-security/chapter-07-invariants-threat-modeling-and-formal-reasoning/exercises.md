# 第 7 章练习题

## 概念题

1. 为什么测试很多，并不自动等于系统安全？
2. 什么是不变量？它和普通功能测试关注点有什么差别？
3. 威胁建模在安全分析里主要回答哪些问题？
4. 为什么说形式化思维能减少“我觉得应该没事”的空间？

## 分析题

1. 某协议所有 happy path 测试都通过，但依旧存在资产会计问题。请分析这更像是不变量缺失还是测试数量不足。
2. 某团队知道很多漏洞案例，但从不写系统资产与权限地图。请分析这会带来什么盲区。
3. 一个协议高度依赖外部价格输入，但威胁模型里几乎没写外部依赖假设。请分析风险。

## 代码题

1. 为一个简单 Vault 写三条核心不变量。
2. 给一个合约系统写一页最小威胁模型，覆盖攻击者目标、能力和关键资产。
3. 审查你自己的一个项目，找出两条“目前只靠口头理解维持”的安全假设。
