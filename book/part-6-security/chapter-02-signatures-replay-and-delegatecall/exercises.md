# 第 2 章练习题

## 概念题

1. 为什么签名重放问题本质上是上下文绑定不足？
2. `nonce` 和 `deadline` 在离线签名里各自解决什么问题？
3. `delegatecall` 为什么比普通 `call` 更危险？
4. 为什么升级代理一定要特别关注实现地址和初始化边界？

## 分析题

1. 某白名单签名只包含 `user` 地址，没有链 ID、合约地址和到期时间。请分析风险。
2. 某代理合约允许管理员随时把 implementation 改成任意地址。请分析这个设计的高危点。
3. 为什么说签名授权和 `delegatecall` 都是在分配权力，而不是只是在执行功能？

## 代码题

1. 设计一个包含 `nonce` 和 `deadline` 的最小签名结构体。
2. 写一个最小代理示例，并说明 `delegatecall` 修改的是谁的状态。
3. 给一个授权流程写失败路径测试，断言重放或过期会被拒绝。
