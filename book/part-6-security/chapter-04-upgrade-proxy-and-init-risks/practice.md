# 第 4 章项目实践

## 实践目标

这次实践要把你从“知道 upgradeable”推进到“能审 upgradeable”。你需要围绕一个真实代理系统，完成一份结构化风险走查。

## 操作步骤

1. 选择一个公开可读的 upgradeable 合约或协议模块。
2. 标记出：
   - proxy 入口
   - implementation 逻辑
   - 初始化函数
   - 升级权限路径
3. 写一页安全分析：
   - 初始化是否安全
   - 升级权限是否合理
   - 是否存在存储兼容风险
   - 一次恶意升级的后果是什么
4. 最后给出 3 到 5 条改进建议。

## 交付物

你应该交付：

1. 一份代理系统结构图
2. 一份安全审查笔记
3. 一份升级风险 checklist
