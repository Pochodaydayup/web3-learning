# 第 3 章项目实践

## 实践目标

这次实践的目标不是让你立刻复现大型攻击，而是训练你用“系统链路”方式描述风险。你要选一个公开的价格操纵或清算异常案例，把它拆成可读、可讲、可复盘的工程材料。

## 操作步骤

1. 选择一个公开事件。优先选有 postmortem、审计补充说明或治理讨论的案例。
2. 写一页“攻击前提”笔记，包含：
   - 协议依赖了什么价格
   - 价格来自哪里
   - 为什么该输入可被短时影响
3. 再画出完整攻击链：
   - 资金从哪里来
   - 哪一步改变了价格或状态
   - 受害协议在哪一步做出了错误判断
   - 攻击者如何落袋收益
4. 最后补一页“如果我是协议方，我会怎么改”：
   - 更换价格源
   - 增加 TWAP
   - 增加上限或冷却时间
   - 收紧敏感参数

## 交付物

你应该交付 3 份内容：

1. 一张攻击流程图
2. 一份 800 到 1500 字的事件复盘
3. 一份防御建议清单，按“立刻可做 / 中期改造 / 结构性重构”三类整理
