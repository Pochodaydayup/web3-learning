# 第 3 章练习题

## 概念题

1. 为什么说 Oracle 不只是“数据接口”，而是协议安全边界？
2. Flash Loan 为什么通常不是漏洞本身，而是漏洞放大器？
3. spot price、TWAP、多源聚合价格各自试图解决什么问题？
4. 为什么 MEV 会让“按用户点击顺序执行”这种直觉失效？

## 分析题

1. 假设一个抵押借贷协议直接读取某个小池子的现货价格来判断清算资格。请你分析这个设计的主要风险，并说明攻击者可能如何组织攻击路径。
2. 某协议已经接入了预言机，但仍在关键函数里允许用户指定非常宽的滑点参数。请分析这是否依然可能暴露 MEV 风险。
3. 如果一个协议宣称“我们用了 Chainlink，所以没有价格攻击风险”，你会继续追问哪些问题？

## 代码题

1. 写一个伪代码版清算函数，标出其中所有依赖价格输入的关键点，并说明每个点应该增加什么防护。
2. 设计一个简单的检查清单，用来审查“基于价格执行 mint / borrow / liquidate”的函数。
3. 画出一次典型的 Flash Loan 价格操纵攻击流程图，要求至少包含资金借入、价格偏移、受害协议执行、还贷结算四步。
