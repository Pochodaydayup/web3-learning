# 第 3 章：Oracle、Flash Loan 与 MEV 风险

很多开发者学安全时，一开始只盯着重入和权限控制，觉得只要把函数边界写严一点，系统就安全了。可一旦进入真实 DeFi 协议世界，你会发现危险并不总藏在单个函数里，很多风险来自“系统依赖了什么价格”“攻击者能瞬间调动多少资金”“你的交易会不会被别人重新排序”。

## 本章要解决什么

本章要解决的是“合约本身看起来没错，但为什么协议还是会被打穿”这个问题。你会把 Oracle、Flash Loan 和 MEV 放进同一张图里理解，因为它们本质上都与外部状态依赖、交易时序和攻击成本结构有关。

## 读完后你应该会什么

- 理解为什么价格输入本身就是一个高风险安全边界
- 知道 Flash Loan 为什么会放大原本就存在的设计漏洞
- 建立对 MEV、抢跑、夹子交易和清算竞争的基本判断
- 在设计协议逻辑时主动检查价格来源、时间窗口和操纵成本

## 本章核心概念

- **Oracle**：把链外或聚合后的外部信息带给链上系统的机制。
- **Flash Loan**：在单笔交易里完成借入、使用和归还的大额无抵押流动性。
- **操纵窗口（Manipulation Window）**：攻击者能够在多大时间或状态范围内扭曲系统输入的机会窗口。
- **MEV**：交易排序和区块构造权所带来的可提取价值。
