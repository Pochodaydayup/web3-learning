# 第 6 章练习题

## 概念题

1. 为什么不同 RPC 对同一条链上数据的读取结果可能在短时间内不一致？
2. 为什么默认读取 `latest` 可能导致一个页面里多个模块彼此“错拍”？
3. 链上状态读取、日志读取和索引器读取各自的同步边界有什么不同？
4. 为什么说多数据源设计的目标不是“永远完全一致”，而是“明确分层”？

## 分析题

1. 某资产页在交易确认后，余额已更新，但明细列表还没变化。请分析这更可能是哪类一致性问题。
2. 某页面连续三次查询同一个仓位信息，返回的 block number 分别是 `120`、`121`、`120`。请分析可能发生了什么。
3. 一条事件在页面上短暂出现后又消失。请从重组、索引滞后和缓存角度分析原因。

## 代码题

1. 设计一个前端读取策略，区分“链上最终确认数据”和“索引聚合展示数据”。
2. 给一个交易详情页写一套状态说明，明确：
   - receipt 来自哪里
   - contract state 来自哪里
   - indexer 列表来自哪里
3. 审查你现有 dApp 的一个页面，写出它最容易产生多源不一致的两个位置和改进方案。
