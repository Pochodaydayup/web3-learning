# 第 6 章项目实践

## 实践目标

这次实践的目标，是把你最熟悉的一个链上页面改造成“多数据源有边界”的版本，而不是把所有读取都混在一起。

## 操作步骤

1. 选择一个有明显状态变化的页面，例如资产页、订单页、投票页或 staking 页。
2. 列出页面当前所有读取来源：
   - 直接 RPC 读状态
   - 查 event 或 receipt
   - 查 indexer / 自建 API
3. 给每个来源标记它的用途：
   - 快速反馈
   - 最终确认
   - 聚合展示
4. 选一组关键字段，尝试让它们基于同一个 block number 读取。
5. 设计一个“索引器延迟时页面怎么解释”的提示方案。
6. 最后写一页复盘，说明：
   - 你现在的页面是否隐含了错误一致性承诺
   - 你准备怎样分层修正

## 交付物

你应该交付：

1. 一份页面读取来源清单
2. 一张多数据源分层图
3. 一份新的状态提示与降级策略说明
