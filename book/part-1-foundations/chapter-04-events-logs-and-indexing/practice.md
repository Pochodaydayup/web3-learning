# 第 4 章项目实践

## 实践目标

这次实践要让你真正看懂事件系统是如何支撑链下可观测性的。你需要围绕一个真实合约，完成一份“状态 + 事件”双视角说明。

## 操作步骤

1. 选择一个熟悉的协议合约，例如 ERC20、NFT 或 staking 合约。
2. 列出它最重要的状态变量，以及与之相关的关键 event。
3. 对任意一笔真实交易，记录：
   - 调用了哪个函数
   - 改了哪些状态
   - 发出了哪些事件
4. 再写一页分析：
   - 哪些信息只存在状态里
   - 哪些信息主要通过事件对外广播
   - 如果链下索引器漏掉这笔事件，如何通过状态重新校准

## 交付物

你应该交付：

1. 一份“状态与事件映射表”
2. 一份真实交易分析笔记
3. 一份 500 到 1000 字的总结，说明你如何理解 event 在系统中的职责
