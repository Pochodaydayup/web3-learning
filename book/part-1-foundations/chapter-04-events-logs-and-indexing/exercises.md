# 第 4 章练习题

## 概念题

1. 为什么说事件更像“发生记录”，而状态更像“当前事实”？
2. receipt、log、topic、`indexed` 之间分别是什么关系？
3. 为什么其他合约不能把历史 event 当成普通状态来读取？
4. 前端为什么不应该长期只依赖事件而不回读状态？

## 分析题

1. 假设一个合约把用户积分只写进 event，不写进 storage。请分析这种设计在链上逻辑和链下同步上会有什么问题。
2. 某协议的前端完全依赖事件更新仓位信息，但偶尔出现 UI 与真实仓位不同步。请分析可能原因。
3. 对 ERC20 `Transfer` 事件来说，为什么 `from` 和 `to` 常常被设为 `indexed`？

## 代码题

1. 找一个你熟悉的合约，列出它最关键的三个 event，并解释每个 event 服务于哪类链下系统。
2. 打开一笔真实交易的 receipt，标记里面的 `status`、`gasUsed` 和 `logs` 分别扮演什么角色。
3. 设计一个简单 staking 合约的事件集合，要求同时考虑前端展示、历史追踪和链下索引需求。
