# 第 1 章延伸阅读

## 必读

- Ethereum 开发者文档中的交易与账户基础部分
- `viem` 或 `ethers` 的合约写入与等待确认文档
- 你常用钱包的签名与交易确认说明

阅读这些资料时，不要只看 API 名字，要刻意记录每一步在整条执行链路里的位置。

## 进阶

- 进一步理解 `nonce`、交易替换、取消交易和 pending 池传播
- 对照浏览器里的 input data 解码一笔真实交易
- 学习 `call`, `estimateGas`, `sendRawTransaction` 这些 RPC 方法分别承担什么角色

## 实战

- 找一笔你曾经遇到过的失败交易，重新复盘失败发生在哪一层
- 给你自己的 dApp 设计一套更细的交易状态机
- 为项目文档补一页“链上交互生命周期说明”，让团队里其他前端也能共享这套心智模型
