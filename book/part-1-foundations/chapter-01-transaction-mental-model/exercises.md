# 第 1 章练习题

## 概念题

1. 为什么“用户已经签名”不能等同于“链上已经成功执行”？
2. `view` 调用和真实交易最本质的执行差异是什么？
3. 为什么 event 很适合给前端做反馈，但不能直接替代主状态？
4. 请用你自己的话解释 `calldata` 在一次外部调用里扮演的角色。

## 分析题

1. 某用户点击“领取奖励”后拿到了交易哈希，但 15 分钟过去仍然没有成功提示。请你从交易生命周期角度列出至少 5 个可能原因，并说明它们更可能属于哪一层。
2. 某个前端在监听到 `Deposit` 事件后立刻把页面余额加上存入金额，但随后用户刷新页面发现余额其实没变。请分析这个系统设计可能有什么问题。
3. 为什么说“RPC 返回成功”并不等于“业务执行成功”？请把“RPC 层成功”和“EVM 层成功”拆开解释。

## 代码题

1. 用 `viem` 或 `ethers` 写一段代码，分别打印一笔交易的：
   - 发送时的 `data`
   - 返回的交易哈希
   - receipt 中的 `status`
   - receipt 中的 `gasUsed`
2. 写一个前端状态枚举，至少覆盖：
   - 等待签名
   - 已广播
   - 已确认
   - 执行失败
3. 选一笔你自己发过的测试网交易，解析它的输入参数，并写一段说明，解释它在链上的完整执行路径。
