# 第 8 章项目实践

## 实践目标

这次实践要把你的一个真实交易入口改造成“有预检查，但不过度承诺”的版本。

## 操作步骤

1. 选择一个真实交互，例如 mint、stake、swap、claim 或 vote。
2. 梳理当前流程里有哪些前置条件：
   - 钱包连接
   - 正确链
   - 输入合法性
   - 余额或 allowance
   - contract simulation
3. 在代码里把这些检查拆成独立阶段，不要塞在一个 `try/catch` 里。
4. 给发送流程增加三个状态提示：
   - 发送前检查通过
   - 交易已广播，等待打包
   - receipt 已返回，按链上结果更新 UI
5. 写一份简短复盘，说明哪些问题能被 preflight 提前发现，哪些问题仍然只能依赖最终链上结果。

## 交付物

你应该交付：

1. 一张交易前检查流程图
2. 一份 preflight checklist
3. 一段发送前后状态提示的 UI 文案设计
