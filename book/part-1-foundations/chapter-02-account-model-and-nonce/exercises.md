# 第 2 章练习题

## 概念题

1. 外部账户和合约账户最核心的差异是什么？
2. nonce 为什么既能约束顺序，又能帮助防止重放？
3. 为什么一笔 pending 交易可能把后续交易也卡住？
4. “加速交易”和“取消交易”在底层上有什么共同点？

## 分析题

1. 某用户连续发送两笔交易，后一笔 gas 更高却迟迟不上链。请从账户顺序和 nonce 角度解释。
2. 某钱包 UI 告诉用户“已取消交易”，但链上其实出现了一笔 0 金额自转账。请解释这种体验是如何实现的。
3. 为什么说 nonce 不是一个“钱包帮你自动填了就不用理解”的字段？

## 代码题

1. 用你熟悉的库读取某地址当前 nonce，并把它和最近一笔交易的 nonce 对上。
2. 写一段脚本模拟同 nonce 替换交易的交易对象差异。
3. 找两笔连续 nonce 的真实交易，写一页顺序分析说明。
