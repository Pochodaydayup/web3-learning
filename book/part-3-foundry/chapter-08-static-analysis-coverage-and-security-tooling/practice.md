# 第 8 章项目实践

## 实践目标

这次实践要把你的一个 Foundry 仓库升级成“测试之外还有第二、第三道防线”的版本。

## 操作步骤

1. 选择一个已有的合约项目。
2. 跑一次当前测试，记录：
   - 有多少测试
   - 是否包含 fuzz 或 invariant
   - 是否有 build warning
3. 给仓库补一套最小质量流水线：
   - 编译检查
   - 测试
   - 覆盖率
   - 静态分析
4. 从报告中挑三条最值得处理的问题，写出：
   - 这是代码问题、测试问题还是配置问题
   - 它为什么重要
   - 是否应作为 CI 阻断项
5. 最后整理一份团队说明，解释为什么“测试全绿”仍然不够。

## 交付物

你应该交付：

1. 一份 CI 命令清单
2. 一份工具报告复盘
3. 一张质量防线分层图
