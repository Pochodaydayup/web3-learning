# 第 4 章：调试、Trace 与 Gas 报告

当你开始频繁写测试和部署脚本后，很快就会发现，真正拉开效率差距的，不是会不会跑命令，而是出错时你能不能迅速看懂执行过程。本章就是要把 Foundry 里的调试能力讲清楚。

## 本章要解决什么

本章要解决的是“测试失败时我只看到 revert，却不知道怎么往下查”的问题。你会把 trace、verbose 输出、断言失败、gas report 和最小复现场景串成一条更成熟的调试路径。

## 读完后你应该会什么

- 用 trace 读懂一次失败调用到底卡在哪一层
- 区分断言失败、权限失败、状态错误和外部调用错误
- 用 gas report 观察成本变化，而不是靠感觉猜
- 建立“先缩小范围，再读执行细节”的调试习惯

## 本章核心概念

- **Trace**：交易或测试执行过程中逐步展开的调用与操作轨迹。
- **Revert Data**：执行失败时向外层返回的错误编码信息。
- **Debugger**：帮助你逐步观察执行过程和状态变化的调试工具。
- **Gas Snapshot**：记录关键路径 gas 消耗并用于回归比较的基准数据。
