# 第 6 章练习题

## 概念题

1. 为什么测试通过和脚本可运行，并不等于已经具备生产发布能力？
2. 一次完整的链上发布至少应该记录哪四层核心信息？
3. 为什么多网络发布的关键不是“换 RPC”，而是显式管理差异？
4. 链上“回滚预案”为什么通常不是简单撤销部署？

## 分析题

1. 某团队把 proxy 地址记录在文档里，但 implementation 地址和初始化参数只留在部署人本地。请分析风险。
2. 某项目在测试网表现正常，上主网后却因依赖地址错误导致异常。请分析这个问题暴露了哪些发布流程缺口。
3. 发布后发现权限账户配置错误，但合约已上线。请从损害控制视角分析应对动作。

## 代码题

1. 为一个小型协议设计发布清单，分成“发布前检查”和“发布后验证”两部分。
2. 写一份地址登记模板，至少包含网络、角色、版本、部署时间和参数摘要。
3. 审查你自己的部署流程，找出三处最缺少正式产物沉淀的地方。
