# 第 2 章练习题

## 概念题

1. 单元测试、fuzz 和 fork 各自更适合覆盖什么问题？
2. 为什么说 fork 的价值不在“更高级”，而在“更真实”？
3. 为什么很多安全边界只测 happy path 是不够的？
4. 什么时候 fuzz 比手工枚举更有价值？

## 分析题

1. 某项目所有测试都只在本地 mock token 上运行。请分析它上线前可能遗漏哪些问题。
2. 某同学每次加功能都先写 fork test，导致开发效率很低。请分析他的验证顺序问题。
3. 某函数参数空间很大，但团队只测了 3 个样例。请说明 fuzz 在这里的意义。

## 代码题

1. 给一个最小合约分别补：
   - 单元测试
   - 1 个 fuzz 测试
   - 1 个 fork 思路测试
2. 写一个 fuzz 测试验证某个不变量。
3. 创建一个 fork，并在真实 token 或协议地址上读取状态做断言。
