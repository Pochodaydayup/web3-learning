# 第 2 章：Forge Test、Fuzz 与 Fork

当你已经能把一个 Foundry 项目跑起来之后，真正决定工程质量的，是你怎么用测试去覆盖假设。单元测试、fuzz 和 fork 不是三种彼此独立的技巧，而是三种覆盖不同风险面的验证方式。

## 本章要解决什么

本章要解决的是“我知道要写测试，但不知道该用哪种测试方式覆盖哪类问题”。你会建立一个更稳定的验证分层视角。

## 读完后你应该会什么

- 区分单元测试、fuzz 和 fork 的主要使用场景
- 知道什么时候测试 happy path，什么时候专门测失败路径
- 理解为什么真实协议交互场景经常离不开 fork
- 用更结构化的方式设计测试策略
