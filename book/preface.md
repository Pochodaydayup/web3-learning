# 前言

## 为什么把它写成一本书

很多 Web3 学习资料都很碎。前端接入、Solidity 语法、Gas 优化、安全事故、协议源码、底层原理，各自都有人讲，但很少有资料会把它们组织成一条适合工程师逐层向下学习的主线。这本书的目标不是再堆一份资料表，而是给已经做过 dApp 的前端开发者一条可长期复用的学习路径。

## 适合谁读

- 已经做过钱包连接、签名、合约读写和事件监听
- 已经知道 RPC、ABI、链 ID、Gas fee、ERC20、NFT 这些概念在应用层的作用
- 想进一步掌握 Solidity、Foundry、EVM、Gas、安全、DeFi 和协议源码阅读

## 不适合谁

- 还没有写过任何链上交互页面
- 只想快速找到一段“接钱包就能跑”的复制粘贴代码
- 期待一本把所有链、所有协议、所有研究方向都讲完的百科全书

## 这本书怎么读

这本书的最佳用法是“主线阅读 + 输出练习 + 支线深入”。

1. 主线阅读：先读卷一到卷六，把交易、状态、执行和安全这条工程主线吃透。
2. 输出练习：每章做练习题、最小项目实践，确保不是只停留在阅读层。
3. 支线深入：在每章的 `reading.md` 中选一到两个材料继续往下挖。

不要试图一开始就把密码学、共识、MEV、L2、桥和复杂 DeFi 一起学完。先建立解释力，再扩大范围。
