# 第 5 章练习题

## 概念题

1. 为什么说套利经常是协议状态与市场重新对齐的力量？
2. MEV 为什么会让“交易按用户意图顺序执行”变得不可靠？
3. 为什么滑点、清算奖励、时间窗口都属于博弈参数？
4. 为什么公开可观察性本身就是协议设计约束？

## 分析题

1. 某个 swap 场景很容易被夹击，通常暴露了哪些参数或路径问题？
2. 为什么有些协议设计在白板上看起来合理，放到链上公开环境里却会被迅速利用？
3. 如果一个清算逻辑利润空间非常明确，会给系统带来哪些竞争行为？

## 代码题

1. 选一个真实 DeFi 操作路径，分析它最可能暴露的公开博弈机会。
2. 画一张“用户提交 -> 公开观察 -> 重排尝试 -> 最终结果”的 MEV 流程图。
3. 给一个协议动作设计用户保护参数，并说明它要缓解什么风险。
