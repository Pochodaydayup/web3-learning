# 第 3 章项目实践

## 实践目标

这次实践的目标，是让你能用自己的语言解释一个借贷仓位为什么安全、为什么危险、为什么会被清算。你要把协议界面中的指标和链上状态逻辑对应起来。

## 操作步骤

1. 选择一个借贷协议界面，例如 Aave 测试网或其他公开 demo。
2. 记录一个具体仓位的几个字段：
   - 抵押资产及数量
   - 借款资产及数量
   - 健康度或同类风险指标
   - 当前价格来源说明
3. 画出仓位变化图：
   - 抵押资产价格上涨
   - 抵押资产价格下跌
   - 借款增加
   - 偿还部分债务
4. 用自己的话解释每种变化为什么会影响健康度。
5. 如果条件允许，写一个最小合约或脚本，模拟“抵押 -> 借款 -> 价格变化 -> 清算”的数据流。

## 交付物

你需要交付：

1. 一份仓位分析笔记
2. 一张健康度变化图
3. 一份最小脚本或伪代码，展示借贷协议中的核心状态流转
