# 第 4 章项目实践

## 实践目标

这次实践要让你把“可升级合约”从概念听说，变成可以解释其风险边界的工程对象。你不一定要立即完整部署代理，但要能画出完整结构。

## 操作步骤

1. 选择一个简单业务，例如 `Vault`、`Bank` 或 `Staking`。
2. 先写出普通版本，再改写成 upgradeable 版本的结构草图。
3. 记录这两个版本的差别：
   - 初始化方式
   - 继承库差异
   - 状态变量规划
   - 管理员权限边界
4. 最后再写一页风险笔记：
   - 哪些错误会导致初始化被劫持
   - 哪些变更会破坏存储布局
   - 哪些权限需要多签或治理保护

## 交付物

你应该交付：

1. 一份普通版与升级版的结构对比笔记
2. 一张 proxy / implementation / state 的关系图
3. 一份升级前检查清单
