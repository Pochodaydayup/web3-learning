# 第 4 章练习题

## 概念题

1. 为什么可升级合约常常不用 constructor，而改用 initializer？
2. proxy 和 implementation 在职责上分别承担什么？
3. 为什么存储布局在升级场景下格外危险？
4. UUPS 和 Transparent Proxy 的核心共同点是什么？

## 分析题

1. 某合约升级后余额数据异常，请分析为什么存储布局变化可能是根因。
2. 如果一个 implementation 没有正确锁住初始化入口，攻击者可能获得什么能力？
3. 为什么“能升级”本身并不能说明升级系统设计是成熟的？

## 代码题

1. 打开一个 OpenZeppelin Upgradeable 合约，对比它和普通版本在 constructor、变量声明和初始化逻辑上的差异。
2. 设计一个最小可升级 Bank 合约的初始化流程，说明哪些状态必须在 `initialize()` 中配置。
3. 写一份升级检查清单，至少包含存储兼容、权限、初始化和测试四类检查项。
