# 第 4 章：升级模式与初始化

当你开始读真实协议或准备写可长期演进的合约时，很快就会遇到一个问题：链上代码一旦部署就不可直接修改，那大型协议为什么还能升级？这一章会把这个问题拆开讲清楚。

## 本章要解决什么

本章要解决的是“升级代理为什么存在、它到底改了什么、又带来了什么新风险”这个问题。你会理解 proxy、implementation、initializer、存储布局约束，以及为什么升级不是一个简单的工程便利，而是一整套额外纪律。

## 读完后你应该会什么

- 解释为什么可升级合约需要绕开 constructor，转而使用 initializer
- 理解 proxy 与 implementation 的职责分离
- 知道存储布局为什么在升级场景下极度敏感
- 对 Transparent Proxy、UUPS 这类模式建立最基础判断
