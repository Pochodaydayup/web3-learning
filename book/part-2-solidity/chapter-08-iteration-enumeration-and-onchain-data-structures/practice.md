# 第 8 章项目实践

## 实践目标

这次实践要做的是：为一个真实合约场景设计一版“状态能执行、查询也现实”的数据结构方案。

## 操作步骤

1. 选择一个小场景，例如：
   - Todo 列表
   - 投票系统
   - Staking 名单
   - NFT 持仓展示
2. 列出这个系统的三类读取需求：
   - 按 key 精确查询
   - 列出全部实体
   - 做统计或分页
3. 设计两套方案并对比：
   - 纯链上可枚举
   - 链上主状态 + 事件 + 链下索引
4. 写出每套方案的优缺点，重点比较：
   - 写入 gas
   - 删除复杂度
   - 前端查询体验
   - 长期维护成本
5. 最后选定一套你认为更现实的结构，并写下原因。

## 交付物

你应该交付：

1. 一张数据结构对比表
2. 一份状态字段与索引字段清单
3. 一段对最终方案的工程说明
