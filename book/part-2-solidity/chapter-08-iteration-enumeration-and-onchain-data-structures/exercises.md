# 第 8 章练习题

## 概念题

1. 为什么 `mapping` 天然不适合列出全部 key？
2. 为什么说可枚举性往往是在用写成本换读能力？
3. event 和链上主状态分别适合承担什么职责？
4. 在什么情况下，链下索引比链上枚举更合理？

## 分析题

1. 一个 Staking 合约需要展示“所有参与过的地址”。请分析它是更适合链上维护数组，还是链下靠事件索引。
2. 某 NFT 合约为了做分页，把每次 mint 都写入多个辅助结构，导致 gas 明显上升。请分析这种设计可能的问题。
3. 某系统把排行榜逻辑完全放在合约里维护。请分析它在复杂度、费用和升级上的代价。

## 代码题

1. 用 `mapping + array + seen` 设计一个最小用户注册索引结构，并解释删除时会遇到什么问题。
2. 审查你自己的一个合约状态设计，指出哪份数据其实更适合链下索引。
3. 为一个投票或 Staking 合约写一版“链上主状态 / 链下查询模型”拆分方案。
