# 第 5 章练习题

## 概念题

1. 为什么很多合约 bug 本质上是状态迁移 bug，而不是语法 bug？
2. 状态机思维如何帮助你设计权限边界？
3. 为什么非法路径和失败路径也必须被视为状态设计的一部分？
4. 状态机思维为什么不等于“必须写 enum”？

## 分析题

1. 某个合约在“结束后仍可继续调用核心函数”。请分析这属于哪类状态机设计问题。
2. 如果一个系统的生命周期被拆散在很多布尔值里，会带来哪些阅读和测试问题？
3. 为什么状态显式化通常会让测试设计更自然？

## 代码题

1. 选一个你写过的合约，把它的主要状态和允许转换整理成表格。
2. 为一个投票或拍卖合约设计状态机，并说明每个阶段允许的动作。
3. 在一个现有合约里找出最容易出现非法状态转换的地方，并写出改进思路。
