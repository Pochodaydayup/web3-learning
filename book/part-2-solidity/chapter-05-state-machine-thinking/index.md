# 第 5 章：合约设计中的状态机思维

到了这一卷的最后一章，你已经会看状态、会看函数边界、会看标准实现和升级纪律。现在要把这些重新收束成一个更高层的设计能力：把合约当成状态机，而不是函数集合。

## 本章要解决什么

本章要解决的是“为什么很多合约逻辑一开始看起来能跑，后面却越来越难维护、越来越容易漏边界”这个问题。你会学会从状态、阶段、允许转换和非法路径的角度设计合约。

## 读完后你应该会什么

- 把一个合约抽象成有限状态和允许转移的系统
- 识别哪些 bug 本质上是非法状态转换
- 设计更清晰的生命周期与权限边界
- 用状态机视角提高合约可读性与可维护性
