# 第 2 章项目实践

## 实践目标

本章实践的目标，是给一个小型合约做一次“接口层重构”。不改业务目标，只优化入口、事件和错误表达，让合约更像一个清晰的工程系统。

## 操作步骤

1. 选择你已经写过的 `Todo`、`Bank` 或 `Voting` 合约。
2. 列出所有外部函数，判断它们是否真的都应该对外暴露。
3. 为关键状态变化设计事件。
4. 把至少 2 个字符串错误改成 custom error。
5. 如果有重复权限或前置判断，抽象成 modifier。
6. 为这些边界调整补测试。

## 交付物

1. 一份接口清单。
2. 一版重构后的合约代码。
3. 一组新增测试。
4. 一页复盘，解释哪些边界最值得前置定义。
