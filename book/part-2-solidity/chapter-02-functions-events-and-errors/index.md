# 第 2 章：Function、Event、Error 与边界设计

学完状态和数据区域之后，下一步最值得补上的，是合约怎样把能力、结果和失败暴露给外部世界。很多 Solidity 初学者以为函数、事件和错误只是语法点，但它们其实是在定义系统边界。

## 本章要解决什么

本章要解决的是“我会写函数，但不知道怎样把接口设计得清晰、可靠、可消费”。你会把 function、visibility、event、custom error 和 modifier 看成一整套边界表达工具。

## 读完后你应该会什么

- 判断某个函数为什么应该是 `external`、`public` 或 `internal`
- 区分哪些结果应该落状态，哪些应该发 event
- 理解 custom error 为什么比字符串报错更适合工程化使用
- 把函数签名、事件设计和错误表达放进统一接口思维里
