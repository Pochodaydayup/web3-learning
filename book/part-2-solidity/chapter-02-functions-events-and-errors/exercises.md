# 第 2 章练习题

## 概念题

1. `external` 和 `public` 在接口语义上最核心的差异是什么？
2. event 的职责为什么和主状态不同？
3. custom error 为什么比字符串更适合工程化处理？
4. modifier 什么时候值得用，什么时候不值得滥用？

## 分析题

1. 某合约把所有函数都写成 `public`。请分析这种做法的风险。
2. 某前端团队完全依赖事件更新 UI，从不回读主状态。请说明这种设计的局限。
3. 某合约所有失败都用 `"error"` 一个字符串。请分析这种错误表达对测试和前端会有什么影响。

## 代码题

1. 给一个最小 `Bank` 合约补上：
   - `Deposited` 事件
   - `Withdrawn` 事件
   - 至少 2 个 custom error
2. 把一个重复权限判断逻辑抽成 modifier。
3. 写测试断言某个 custom error 被正确抛出。
