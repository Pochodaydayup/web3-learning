# 第 3 章练习题

## 概念题

1. 为什么说标准代币首先是一种接口契约？
2. OpenZeppelin 替你复用的核心价值是什么？
3. 为什么继承标准实现不等于不思考？
4. 扩展标准实现时最该警惕什么？

## 分析题

1. 某团队决定自己手写完整 ERC20，只因为“这样更懂”。请分析这在工程上可能带来的问题。
2. 某 NFT 合约继承了标准实现，但改坏了核心事件语义。请解释这为什么危险。
3. 为什么说标准实现一旦被错误 override，风险有时比从零写还隐蔽？

## 代码题

1. 用 OpenZeppelin 写一个最小 ERC20。
2. 给它补一个简单权限扩展，并说明风险点。
3. 把你自己的最小代币实现和 OpenZeppelin 版本做一页对比笔记。
