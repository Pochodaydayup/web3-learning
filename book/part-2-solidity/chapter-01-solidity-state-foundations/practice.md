# 第 1 章项目实践

## 实践目标

本章的实践目标是自己设计一个小型状态系统，而不是复刻一个模板。推荐项目是 `Bank` 或 `TodoList`，因为它足够小，但已经能暴露出状态、权限、event 和错误表达这几类基础问题。

## 操作步骤

1. 先写一页状态设计说明，回答：
   - 核心状态有哪些
   - 哪些信息必须持久化
   - 哪些信息只是链下展示辅助
2. 开始编写合约，优先实现最小可用版本：
   - `deposit` / `withdraw` 或 `create` / `toggle`
   - 必要的 `event`
   - 基础错误处理
3. 在代码完成后，逐条检查：
   - 哪些参数应该是 `calldata`
   - 哪些结构应该在 `memory` 中临时处理
   - 是否有不必要的状态写入
4. 给合约补一组最小测试，至少覆盖：
   - 正常路径
   - 权限失败
   - 余额不足或状态非法时的回滚
5. 最后写一页复盘，说明如果让你重写一次，哪些状态结构会调整。

## 交付物

1. 一份状态设计说明。
2. 一个可编译的小型 Solidity 合约。
3. 一组覆盖成功路径和失败路径的测试。
4. 一份复盘说明，重点解释你对 `storage`、`memory`、`calldata` 的使用判断。
